    02 · MDP 与网格世界（Gridworld）：把"未来"引入 RL¶

    从这一章开始，RL 和 Bandit 最大的区别出现了：你现在的动作会改变未来遇到的状态。

    核心概念: 状态转移、贝尔曼方程、价值函数 图解: figures/02_bellman_intuition.png

    0. Bellman 直觉：价值函数为什么能递推¶

    flowchart LR
  S[当前状态 s] -->|做动作 a| R[得到奖励 r]
  R --> Sp[到达下一状态 s']
  Sp -->|未来继续获得价值| V2[折扣后的未来价值 γ·V(s')]
  R --> V1[现在价值的一部分]
  V1 --> V["V(s) = 期望 (r + γ·V(s'))"]
  V2 --> V

    1. 生动案例：通勤路线选择¶

    你每天从家到公司：

    路线 特点 风险
    高速 通常快 可能堵车
    地面 更稳定 可能慢

    你今天选的路，会影响你到达某个路口的时间，从而影响下一段路的拥堵。

    这就是"状态转移"。Bandit 没有这件事。

    2. MDP 五件套（其实就 3 个核心）¶

    MDP（马尔可夫决策过程）通常写成 (S, A, P, R, γ)：

    符号 名称 说明 例子
    S 状态 世界的"当前摘要" 机器人在哪个格子
    A 动作 你能做的选择 上/下/左/右
    P 转移 做了动作后，世界怎么变 向右走 90% 成功
    R 奖励 做完得到的反馈 到终点 +1，每步 -0.01
    γ 折扣 多重视未来 γ=0.9 表示未来打 9 折

    真正核心：

    在状态 s 选择动作 a，环境给奖励 r，并转移到下一状态 s'。

    3. 什么是"马尔可夫"（Markov）性质¶

    直觉版：

    只要你知道"当前状态"，预测未来就不需要更久远的历史。

    例子：

    场景 状态包含 是否马尔可夫
    机器人导航 位置 + 速度 ✅ 是
    机器人导航 只有位置 ❌ 否（缺速度）
    棋类游戏 当前棋盘 ✅ 是
    对话系统 最后一句话 ❌ 否（需要上下文）

    当状态不够时，就变成 POMDP（部分可观测），那是更高级话题。

    4. 网格世界（Gridworld）为什么是最好的入门环境¶

    Gridworld 是一个二维格子地图：

    ┌───┬───┬───┬───┬───┐
│ S │   │   │   │ G │
├───┼───┼───┼───┼───┤
│   │ █ │   │ █ │   │
├───┼───┼───┼───┼───┤
│   │   │   │   │   │
└───┴───┴───┴───┴───┘

S = 起点，G = 终点，█ = 墙

    规则： - 状态 = 你在哪个格子 - 动作 = 上下左右 - 终点奖励 +1 - 每步小惩罚 -0.01（鼓励更短路径）

    优点： - ✅ 可视化强、直觉强 - ✅ 能明确感受到"延迟奖励" - ✅ 非常适合理解价值函数与 Bellman 递推

    5. 价值函数：用"打分"来规划未来¶

    状态价值 V(s)¶

    从状态 s 出发，如果你按某个策略走，长期平均能拿多少分

    动作价值 Q(s,a)¶

    在 s 先做 a，然后继续按策略走，长期平均能拿多少分

    为什么 Q 更常用？

    因为它直接回答"现在该做哪个动作"。

    函数 输入 输出 用途
    V(s) 状态 价值 评估状态好坏
    Q(s,a) 状态 + 动作 价值 选择最佳动作

    6. Bellman 直觉（很重要！）¶

    Bellman 的核心就是一句话：

    现在的价值 = 现在的奖励 + 折扣后的下一步价值

    用公式表示： 

    V(s) = E[r + γ·V(s')]

    当你理解这句话，后面的 Q-learning/DQN/PPO 的很多式子都会"看起来合理"。

    7. 终止任务 vs 持续任务¶

    类型 特点 例子
    终止任务 (episodic) 到达终点/失败就结束 迷宫、游戏一局
    持续任务 (continuing) 没有明确终点 流量分配、在线控制

    折扣 γ 在持续任务里通常更重要。

    8. 代码与实验¶

    文件： - code/gridworld/gridworld_env.py — 环境定义 - code/gridworld/train_q_learning.py — 训练脚本

    运行： 

    python3 code/gridworld/train_q_learning.py

    你应该看到： - 回合平均步数逐渐减少 - 回合总回报逐渐增加 - 最后输出一个箭头网格策略，箭头大多指向终点

    9. 小练习¶

    1. 修改步数惩罚：把 step_cost 从 -0.01 改成 0，会发生什么？

    2. 提示：智能体可能"磨蹭"，不急于到终点

    3. 添加障碍物：在网格里加墙（不可走格子），最短路径会如何变化？

    4. 改变折扣率：把 γ 从 0.9 改成 0.5，智能体变得更"短视"吗？

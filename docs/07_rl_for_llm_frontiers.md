# 07 · LLM 里的 RL：RLHF/RLAIF/DPO/GRPO 与前沿趋势（通俗版）

LLM 训练通常分三段：
1. **预训练**（大量文本，自监督 next-token）
2. **指令微调 SFT**（人写的指令-回答）
3. **对齐**（让回答更符合人类偏好/安全/有用）

第 3 段里，RL 常常登场。

## RLHF 是什么？（一句话）
**RLHF（Reinforcement Learning from Human Feedback）**：
- 人给偏好（A 比 B 更好）
- 训练一个“奖励模型”RM：给回答打分
- 用 RL（常见 PPO）训练语言模型，让它输出更高奖励的回答

直觉：
- 语言模型 = 策略 π
- 生成一句话/一段回答 = 一串动作
- 奖励模型给整体回答打分 = episodic reward

## 现在更流行的替代/补充：DPO / RLAIF
### DPO（Direct Preference Optimization）
很多团队发现：
- 不一定非要显式跑 PPO
- 可以用一个更像“对比学习/分类”的目标直接把“偏好”灌进模型

DPO 的直觉：
> 让模型对“更好回答”的概率更高，对“更差回答”的概率更低，同时不偏离参考模型太远。

优点：
- 工程简单、稳定
- 不需要在线采样那么复杂的 RL 循环

### RLAIF（AI Feedback）
人类标注贵 → 用更强的模型当裁判/老师生成偏好数据。

关键点：
- 质量控制（防止老师胡说）
- 混合少量高质量人类数据做锚

## 近期趋势关键词（你可以当作路线图）
- **Process Reward Model (PRM)**：不仅给最终答案打分，也给“思考过程/中间步骤”打分
- **Verifier / Critic**：训练判别器验证答案是否正确（数学/代码/检索）
- **Online vs Offline RL**：偏好数据能否迭代更新？会不会出现分布偏移？
- **GRPO / Group-based 方法**：用一组采样的相对比较/归一化优势来训练（减少对价值网络依赖）
- **Self-play / Debate / Constitutional AI**：让模型彼此博弈或用规则约束来产生更好监督

## 一个非常实用的工程直觉
LLM 对齐里，很多“RL 组件”在做两件事：
1. **让模型更偏向输出人喜欢的模式**（礼貌、结构化、少幻觉）
2. **控制训练不要跑飞**（KL 约束/参考模型/裁剪）

你可以把“KL 约束”理解成：
> 别变得太不像原来的自己，否则会崩。
